# FAQ

### Project 3를 구현하려면 Project 2가 동작하는 상태여야 하나요?

네.

### 페이지 폴트를 처리한 뒤에는 프로세스를 어떻게 다시 실행하나요?

`page_fault()`에서 반환하면 현재 사용자 프로세스의 실행이 재개됩니다. 그러면
instruction pointer가 가리키는 명령을 다시 시도합니다.
왜 사용자 프로세스는 가끔 stack pointer 위쪽에서 페이지 폴트를 내나요?
스택 확장 테스트를 보면, 사용자 프로그램이 현재 stack pointer보다 위쪽
주소에서 페이지 폴트를 일으키는 것을 볼 수 있습니다.

### 가상 메모리 시스템이 데이터 세그먼트 확장도 지원해야 하나요?

아니요. 데이터 세그먼트의 크기는 linker가 결정합니다. Pintos에는 여전히
동적 할당이 없으며(사용자 수준에서는 메모리 매핑 파일을 이용해 이를 "흉내"
낼 수는 있습니다), 데이터 세그먼트 확장을 지원하더라도 잘 설계된 시스템에
추가되는 복잡도는 크지 않을 것입니다.

### 페이지 프레임을 할당할 때 왜 `PAL_USER`를 사용해야 하나요?

`palloc_get_page()`에 `PAL_USER`를 전달하면 주 커널 풀 대신 사용자 풀에서
메모리를 할당합니다. 사용자 풀의 페이지가 부족하면 사용자 프로그램들이 paging
하게 될 뿐이지만, 커널 풀의 페이지가 부족해지면 많은 커널 함수가 메모리를
얻어야 하므로 여러 실패가 발생하게 됩니다. 원한다면 `palloc_get_page()`
위에 다른 할당기를 올려서 사용할 수도 있지만, 그 함수가 근본적인 할당
메커니즘이어야 합니다.
또한 `-ul` 커널 커맨드라인 옵션을 사용해 사용자 풀의 크기를 제한할 수 있으므로,
다양한 사용자 메모리 크기에서 VM 구현을 쉽게 테스트할 수 있습니다.
