# 07_cow

## Copy-on-write (추가)

**Pintos에 copy-on-write 메커니즘을 구현하세요.**

Copy-on-write는 같은 물리 페이지 인스턴스를 함께 사용함으로써 복제 작업을 더
빠르게 수행하게 해 주는 자원 관리 기법입니다. 어떤 자원을 여러 프로세스가
사용한다면, 일반적으로는 충돌이 일어나지 않도록 각 프로세스가 그 자원의
복사본을 따로 가져야 합니다. 하지만 그 자원이 수정되지 않고 읽기만 된다면,
물리 메모리에 여러 복사본을 둘 필요는 없습니다.

예를 들어 새 프로세스가 `fork`를 통해 생성된다고 가정합시다. 자식은 부모의
자원을 상속받기 위해 그 데이터를 자신의 가상 주소 공간에 복제해야 합니다.
보통 가상 메모리에 내용을 추가하려면 물리 페이지를 할당하고, 프레임에 데이터를
기록하고, 페이지 테이블에 virtual->physical 매핑을 추가해야 합니다. 이런
단계들은 상당한 시간이 들 수 있습니다.

하지만 copy-on-write 기법에서는 새 자원 복사본을 위해 새로운 물리 페이지를
할당하지 않습니다. 엄밀히 말하면 그 내용은 이미 물리 메모리에 존재하기
때문입니다. 따라서 자식 프로세스의 페이지 테이블에 virtual->physical 매핑만
추가하면 됩니다. 이제 그 가상 주소는 자식의 메모리 공간 안에 있게 됩니다.
그 결과 부모와 자식은 같은 물리 페이지에 있는 같은 데이터에 접근하게 됩니다.
그럼에도 두 프로세스는 서로 다른 가상 주소 공간을 통해 격리되어 있으며, 같은
프레임을 참조하고 있다는 사실은 OS만 알고 있습니다. 공유 자원의 내용을
프로세스 중 하나가 수정하려고 할 때에만, 그 프로세스를 위해 새 물리 페이지에
별도의 복사본을 만듭니다. 즉, 실제 복사 작업은 첫 번째 쓰기 시점까지
미뤄집니다.

즉, OS는 copy-on-write 페이지에 대한 쓰기 시도를 감지할 수 있어야 합니다.
이를 위해 OS는 "write-protect" 메커니즘을 사용합니다. 아이디어는 단순합니다.
쓰기 접근에 대해 페이지 폴트가 나도록 만드는 것입니다. 이는 메모리 관리
시스템의 지원을 받아, write-protected 페이지를 모두 쓰기 불가능하게 표시하는
것만으로 쉽게 구현할 수 있습니다.

여러분은 **​`fork`​**​**에 대해서만 copy-on-write를 구현하면 됩니다.**  자식
프로세스가 부모 프로세스로부터 자원을 상속받을 때, 자식이 그 내용을 수정하려
하기 전까지는 부모와 자식이 같은 물리 데이터를 참조하도록 할 수 있습니다.
**모든 write-protected 페이지는 축출 대상이 될 수 있습니다.**

copy-on-write에 대해서는 기본 테스트 케이스만 제공합니다. 가능한 모든 경우를
고려해야 합니다(작은 힌트 하나를 주자면, file-backed page의 공유도 구현해야
합니다). 이 추가 프로젝트의 채점은 hidden test case로도 진행됩니다.
