# 메모리 할당

Pintos에는 두 가지 메모리 할당자가 있습니다. 하나는 페이지 단위로 메모리를 할당하고, 다른 하나는 임의 크기의 블록을 할당할 수 있습니다.

[]()</a>

## 페이지 할당자

`include/threads/palloc.h`에 선언된 페이지 할당자는 페이지 단위로 메모리를 할당합니다. 보통 한 번에 한 페이지씩 메모리를 할당할 때 가장 자주 사용되지만, 여러 개의 연속된 페이지를 한 번에 할당할 수도 있습니다.

페이지 할당자는 자신이 할당하는 메모리를 커널 풀과 사용자 풀이라는 두 개의 풀로 나눕니다. 기본값으로는 각 풀이 1MB 위쪽의 시스템 메모리를 절반씩 가져가지만, 이 분할은 **ul** 커널 명령줄 옵션으로 바꿀 수 있습니다. 할당 요청은 두 풀 중 하나에서 가져옵니다. 한 풀이 비더라도 다른 풀에는 여전히 여유 페이지가 있을 수 있습니다. 사용자 풀은 사용자 프로세스를 위한 메모리 할당에, 커널 풀은 그 밖의 모든 할당에 사용해야 합니다. 이것이 중요해지는 시점은 project 3부터입니다. 그전까지는 모든 할당을 커널 풀에서 해야 합니다.

각 풀의 사용 현황은 비트맵으로 추적하며, 풀의 각 페이지마다 비트 하나를 사용합니다. n개 페이지를 할당하라는 요청이 들어오면, 비트맵에서 false로 설정된 연속된 비트 n개를 찾아 그 페이지들이 비어 있음을 확인한 다음, 해당 비트를 true로 바꾸어 사용 중이라고 표시합니다. 이것은 "first fit" 할당 전략입니다.

페이지 할당자는 단편화(fragmentation)의 영향을 받습니다. 즉, n개 이상의 여유 페이지가 있더라도 그 사이사이에 사용 중인 페이지가 끼어 있으면 n개의 연속된 페이지를 할당하지 못할 수 있습니다. 실제로 병적인 경우에는 풀의 페이지 절반이 비어 있어도 연속된 2페이지를 할당하지 못할 수 있습니다. 한 페이지짜리 요청은 단편화 때문에 실패하지 않으므로, 여러 개의 연속된 페이지를 요구하는 요청은 가능한 한 줄여야 합니다.

인터럽트 컨텍스트에서는 페이지를 할당할 수 없지만, 해제는 할 수 있습니다.

페이지를 해제하면, 디버깅을 돕기 위해 그 페이지의 모든 바이트를 **0xcc**로 채웁니다([디버깅 팁](06_debugging_tools.md#tips) 참고).

페이지 할당자의 타입과 함수는 아래와 같습니다.

---

```c
void *palloc_get_page (enum palloc_flags flags)
void *palloc_get_multiple (enum palloc_flags flags, size_t page_cnt)
```

> 각각 한 페이지 또는 *page_cnt*개의 연속된 페이지를 얻어 반환합니다. 페이지를 할당할 수 없으면 널 포인터를 반환합니다.
>
> *flags* 인자는 다음 플래그를 조합한 값일 수 있습니다:
>
> - `PAL_ASSERT`
>
>   페이지를 할당할 수 없으면 커널을 패닉 상태로 만듭니다. 이는 커널 초기화 중에만 적절합니다. 사용자 프로세스 때문에 커널이 패닉 상태가 되게 해서는 안 됩니다.
> - `PAL_ZERO`
>
>   할당된 페이지를 반환하기 전에 모든 바이트를 0으로 초기화합니다. 이 플래그가 설정되지 않으면 새로 할당된 페이지의 내용은 예측할 수 없습니다.
> - `PAL_USER`
>
>   페이지를 사용자 풀에서 가져옵니다. 이 플래그가 설정되지 않으면 페이지는 커널 풀에서 할당됩니다.

---

```c
void palloc_free_page (void *page)
void palloc_free_multiple (void *pages, size_t page_cnt)
```

> 각각 한 페이지 또는 *page_cnt*개의 연속된 페이지를 *pages*부터 시작해 해제합니다. 모든 페이지는 반드시 `palloc_get_page()` 또는 `palloc_get_multiple()`로 얻은 것이어야 합니다.

## 블록 할당자

`threads/malloc.h`에 선언된 블록 할당자는 임의 크기의 블록을 할당할 수 있습니다. 이 할당자는 앞 절에서 설명한 페이지 할당자 위에 계층적으로 구현되어 있습니다. 블록 할당자가 반환하는 블록은 커널 풀에서 얻어집니다.

블록 할당자는 메모리를 할당할 때 두 가지 서로 다른 전략을 사용합니다. 첫 번째 전략은 1 kB 이하의 블록(페이지 크기의 4분의 1)에 적용됩니다. 이런 할당은 가장 가까운 2의 거듭제곱 단위 또는 16바이트 중 더 큰 쪽으로 올림됩니다. 그런 다음 같은 크기 할당만을 위해 사용되는 페이지에 묶입니다.

두 번째 전략은 1 kB보다 큰 블록에 적용됩니다. 이런 할당은(소량의 오버헤드를 더한 뒤) 크기를 가장 가까운 페이지 단위로 올림하고, 블록 할당자는 그 수만큼의 연속된 페이지를 페이지 할당자에 요청합니다.

어느 경우든, 요청한 할당 크기와 실제 블록 크기의 차이는 낭비됩니다. 실제 운영체제라면 이런 낭비를 최소화하도록 할당자를 세심하게 조정하겠지만, Pintos 같은 교육용 시스템에서는 이것이 중요하지 않습니다.

페이지 할당자에서 페이지를 얻을 수 있기만 하다면, 작은 할당은 항상 성공합니다. 대부분의 작은 할당은 페이지 할당자에서 새 페이지를 받을 필요조차 없습니다. 이미 할당된 페이지의 일부를 사용해 충족되기 때문입니다. 하지만 큰 할당은 항상 페이지 할당자를 호출해야 하며, 이미 앞 절에서 설명했듯이 두 개 이상의 연속된 페이지가 필요한 할당은 단편화 때문에 실패할 수 있습니다. 따라서 코드에서는 큰 할당의 수를 최소화해야 하며, 특히 각 할당이 대략 4 kB를 넘는 경우를 주의해야 합니다.

블록을 해제하면, 디버깅을 돕기 위해 그 블록의 모든 바이트를 **0xcc**로 채웁니다([디버깅 팁](06_debugging_tools.md#tips) 참고).

블록 할당자는 인터럽트 컨텍스트에서 호출할 수 없습니다.

블록 할당자 함수는 아래와 같습니다. 인터페이스는 같은 이름을 가진 표준 C 라이브러리 함수와 동일합니다.

---

```c
void *malloc (size_t size)
```

> 커널 풀에서 길이가 최소 *size* 바이트인 새 블록을 얻어 반환합니다. size가 0이거나 메모리를 사용할 수 없으면 널 포인터를 반환합니다.

---

```c
void *calloc (size_t a, size_t b)
```

> 커널 풀에서 길이가 최소 `a * b` 바이트인 새 블록을 얻어 반환합니다. 블록의 내용은 0으로 채워집니다. a 또는 b가 0이거나 사용할 수 있는 메모리가 부족하면 널 포인터를 반환합니다.

---

```c
void *realloc (void *block, size_t new_size)
```

> *block*의 크기를 *new_size* 바이트로 바꾸려고 시도하며, 이 과정에서 블록이 이동할 수도 있습니다. 성공하면 새 블록을 반환하며, 이 경우 기존 블록에는 더 이상 접근해서는 안 됩니다. 실패하면 널 포인터를 반환하고, 기존 블록은 계속 유효합니다. block이 null인 호출은 `malloc()`과 같습니다. new_size가 0인 호출은 `free()`와 같습니다.

---

```c
void free (void *block)
```

> *block*을 해제합니다. *block*은 반드시 앞서 `malloc()`, `calloc()`, `realloc()` 중 하나가 반환한 것이어야 하며, 아직 해제되지 않은 상태여야 합니다.
