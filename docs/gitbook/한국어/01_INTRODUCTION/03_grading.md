# 03_grading

# 채점

과제는 테스트 결과를 기준으로 채점합니다.

## 테스트

테스트 점수는 우리가 실행하는 테스트 결과를 기준으로 매겨집니다.
각 프로젝트에는 여러 테스트가 있으며, 각 테스트의 이름은 `tests`로 시작합니다.
제출물을 완전히 테스트하려면 프로젝트 `build` 디렉터리에서 `make check`를 실행하세요.
그러면 각 테스트를 빌드하고 실행한 뒤, 각각에 대해 `pass` 또는 `fail` 메시지를 출력합니다. 테스트가 실패하면 `make check`는 실패 이유에 대한 몇 가지 세부 정보도 함께 출력합니다. 모든 테스트 실행이 끝나면 `make check`는 테스트 결과 요약도 출력합니다.
개별 테스트를 하나씩 실행할 수도 있습니다. 특정 테스트 `t`는 자신의 출력을 `t.output`에 기록한 다음, 스크립트가 그 출력을 `pass` 또는 `fail`로 채점하고 판정을 `t.result`에 기록합니다. 단일 테스트를 실행하고 채점하려면 `build` 디렉터리에서 `.result` 파일을 직접 만들어야 합니다. 예를 들어 `make tests/threads/alarm-multiple.result`와 같이 실행하면 됩니다. make가 테스트 결과가 최신 상태라고 말하더라도 다시 실행하고 싶다면, `make clean`을 실행하거나 `.output` 파일을 직접 삭제하세요. 기본적으로 각 테스트는 실행 중간이 아니라 완료 시점에만 피드백을 제공합니다. 원한다면 make 명령행에 `VERBOSE=1`을 지정해 각 테스트의 진행 상황을 볼 수 있습니다. 예를 들어 `make check VERBOSE=1`과 같이 사용합니다.
모든 테스트와 관련 파일은 `pintos/src/tests`에 들어 있습니다. 제출물을 테스트하기 전에 우리는 이 디렉터리의 내용을 깨끗하고 수정되지 않은 복사본으로 교체해 올바른 테스트가 사용되도록 합니다. 따라서 디버깅에 도움이 된다면 일부 테스트를 수정해도 되지만, 실제로는 원본 테스트를 실행합니다.
모든 소프트웨어에는 버그가 있으므로, 우리의 테스트 중 일부에도 결함이 있을 수 있습니다. 테스트 실패가 여러분 코드의 버그가 아니라 테스트의 버그라고 생각되면 알려 주세요.
확인한 뒤 필요하면 수정하겠습니다. 우리가 테스트 스위트를 공개한 점을 악용하려고 하지는 마세요. 여러분의 코드는 우리가 제공한 테스트 케이스에서만 아니라 일반적인 경우에도 올바르게 동작해야 합니다. 예를 들어, 실행 중인 테스트 케이스의 이름에 따라 커널의 동작을 노골적으로 바꾸는 것은 받아들일 수 없습니다. 이런 식으로 테스트를 우회하려는 시도에는 점수를 주지 않습니다. 이 부분에서 자신의 해법이 회색 지대에 있을 수 있다고 생각되면, 먼저 우리에게 문의하세요.

## 제출

아카이브 파일을 만드는 명령을 제공합니다. `threads`, `userprog`, `vm`, `filesys` 디렉터리가 있는 pintos 프로젝트 루트 디렉터리로 이동하세요.
`TEAM=YOUR_TEAM_NUMBER make archive`를 입력합니다(예: `TEAM=5 make archive`). 그러면 같은 디렉터리에 `team5.tar.gz`가 생성됩니다. 이 아카이브 파일을 제출하세요.
**중복 제출은 하지 마세요.**  팀당 한 번만 제출하면 충분합니다.
