# 06_deny_write

### 실행 파일에 대한 쓰기 금지

**실행 파일에 대한 쓰기를 금지하세요.**

실행 중인 파일에 대해서는 쓰기를 막는 코드를 추가하세요. 많은 운영체제가 이렇게 하는 이유는, 디스크 위에서 변경되는 도중인 코드를 프로세스가 실행하려 하면 예측할 수 없는 결과가 나오기 때문입니다. 이는 project 3에서 virtual memory가 구현되고 나면 특히 중요하지만, 지금 해 두어도 해가 되지 않습니다.

열려 있는 파일에 대한 쓰기를 막으려면 `file_deny_write()`를 사용할 수 있습니다. 이후 그 파일에 대해 `file_allow_write()`를 호출하면 다시 쓸 수 있게 됩니다(다른 opener가 그 파일에 대해 여전히 쓰기 금지를 유지하고 있지 않은 한). 파일을 닫아도 쓰기 금지는 해제됩니다. 따라서 프로세스의 실행 파일에 대한 쓰기를 막으려면, 프로세스가 실행 중인 동안 그 파일을 계속 열어 두어야 합니다.
