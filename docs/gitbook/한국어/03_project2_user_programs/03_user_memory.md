# 03_user_memory

### 사용자 메모리 접근

**사용자 메모리 접근을 구현하세요**

syscall을 구현하려면 사용자 가상 주소 공간의 데이터를 읽고 쓸 수 있는 방법을 제공해야 합니다. 인자를 가져올 때는 이 기능이 필요하지 않습니다. 하지만 시스템 콜의 인자로 전달된 포인터가 가리키는 데이터를 읽을 때는 이 기능을 통해 접근해야 합니다. 이는 조금 까다로울 수 있습니다. 사용자가 잘못된 포인터, 커널 메모리를 가리키는 포인터, 혹은 그런 영역 중 하나에 일부만 걸쳐 있는 블록을 제공하면 어떻게 해야 할까요? 이런 경우에는 사용자 프로세스를 종료해서 처리해야 합니다.
