# 07_dup

## 파일 디스크립터 확장 (Extra)

**linux에서처럼 **​**​`stdin`​**​ **, **​**​`stdout`​**​**을 닫을 수 있고 **​**​`dup2`​**​ ** 시스템 콜도 지원하도록 Pintos를 확장하세요**

현재 Pintos 구현에서는 `stdin`과 `stdout`의 fd를 닫는 것이 금지되어 있습니다. 이 extra 요구사항에서는 먼저 Linux처럼 사용자가 `stdin`과 `stdout`을 닫을 수 있게 만들어야 합니다. 즉, `stdin`을 닫으면 그 프로세스는 입력을 읽지 못해야 하고, `stdout`을 닫으면 아무것도 출력하지 못해야 합니다.

다음으로 `dup2` 시스템 콜을 구현하세요.

---

```c
int dup2(int oldfd, int newfd);
```

> `dup2()` 시스템 콜은 파일 디스크립터 `oldfd`를 복제해 `newfd`가 지정한 번호의 파일 디스크립터로 만들고, 성공하면 `newfd`를 반환합니다. `newfd`가 이미 열려 있었다면, 재사용하기 전에 조용히 닫힙니다.
>
> 다음 사항에 유의하세요.
>
> - `oldfd`가 유효한 파일 디스크립터가 아니면 호출은 실패하고(`-1` 반환), `newfd`는 닫히지 않습니다.
> - `oldfd`가 유효하고 `newfd`의 값이 `oldfd`와 같다면, `dup2()`는 아무 일도 하지 않고 `newfd`를 반환합니다.
>
> 이 시스템 콜이 성공적으로 반환된 뒤에는 기존 파일 디스크립터와 새 파일 디스크립터를 서로 바꿔 써도 됩니다. 둘은 **서로 다른 파일 디스크립터**이지만, **같은 open file description을 참조**하므로 *file offset*과 *status flags*를 공유합니다. 예를 들어, 한쪽 디스크립터에서 `seek`를 사용해 file offset을 바꾸면 다른 쪽의 offset도 함께 바뀝니다.

또한 `dup`된 파일 디스크립터는 `fork` 이후에도 그 의미를 유지해야 합니다.

**추가 점수를 받으려면 extra용 테스트 케이스를 모두 통과해야 합니다. 부분 점수는 없습니다.**
