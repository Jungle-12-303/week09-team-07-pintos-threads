# 03_subdirectories

### 하위 디렉터리

**계층적 네임스페이스를 구현하세요.** 
기본 파일 시스템에서는 모든 파일이 하나의 디렉터리에 들어 있습니다. 디렉터리 엔트리가 파일뿐 아니라 다른 디렉터리도 가리킬 수 있도록 수정하세요.

디렉터리도 다른 파일과 마찬가지로 원래 크기를 넘어 확장될 수 있도록 해야 합니다.

기본 파일 시스템은 파일 이름 길이를 14자로 제한합니다. 개별 파일 이름 컴포넌트에 대해서는 이 제한을 유지해도 되고, 원한다면 늘려도 됩니다. 하지만 전체 경로 이름은 14자보다 훨씬 길어질 수 있어야 합니다.

**프로세스마다 별도의 현재 디렉터리를 유지하세요.** 
시작 시에는 루트를 초기 프로세스의 현재 디렉터리로 설정하세요. 한 프로세스가 `fork` 시스템 콜로 다른 프로세스를 시작하면, 자식 프로세스는 부모의 현재 디렉터리를 상속받습니다. 그 이후에는 두 프로세스의 현재 디렉터리가 서로 독립적이어야 하므로, 한쪽이 자신의 현재 디렉터리를 바꾸더라도 다른 쪽에는 영향이 없어야 합니다. (그래서 Unix에서 `cd` 명령은 외부 프로그램이 아니라 셸 내장 명령입니다.)

기존 시스템 콜도 갱신해서, 호출자가 파일 이름을 넘기는 모든 위치에서 절대 경로나 상대 경로를 사용할 수 있도록 하세요. 디렉터리 구분 문자는 슬래시('/')입니다. 또한 Unix와 같은 의미로 특별한 파일 이름 `.`과 `..`도 지원해야 합니다.

`open` 시스템 콜도 디렉터리를 열 수 있도록 갱신하세요. 기존 시스템 콜 중에서는 `close`만 디렉터리를 가리키는 파일 디스크립터를 받아들이면 됩니다.

`remove` 시스템 콜도 일반 파일뿐 아니라 빈 디렉터리(루트 제외)를 삭제할 수 있도록 갱신하세요. 디렉터리는 파일이나 하위 디렉터리(`.`과 `..` 제외)를 포함하지 않을 때만 삭제할 수 있습니다. 프로세스가 열어 둔 디렉터리이거나 프로세스의 현재 작업 디렉터리로 사용 중인 디렉터리를 삭제 가능하게 할지는 여러분이 결정해도 됩니다.
삭제를 허용한다면, 삭제된 디렉터리 안에서 파일(`.`과 `..` 포함)을 열거나 새 파일을 만드는 시도는 금지되어야 합니다.
다음과 같은 새 시스템 콜을 구현하세요.

---

```c
bool chdir (const char *dir);
```

> 프로세스의 현재 작업 디렉터리를 `dir`로 변경합니다. `dir`은 상대 경로일 수도 있고 절대 경로일 수도 있습니다.
> 성공하면 true를, 실패하면 false를 반환합니다.

---

```c
bool mkdir (const char *dir);
```

> `dir`이라는 이름의 디렉터리를 만듭니다. `dir`은 상대 경로일 수도 있고 절대 경로일 수도 있습니다. 성공하면 true를,
> 실패하면 false를 반환합니다. `dir`이 이미 존재하거나, 마지막 요소를 제외한 `dir` 안의 디렉터리 이름 중 하나라도
> 이미 존재하지 않으면 실패합니다. 즉, `mkdir("/a/b/c")`는 `/a/b`가 이미 존재하고 `/a/b/c`는 존재하지 않을 때만 성공합니다.

---

```c
bool readdir (int fd, char *name);
```

> 파일 디스크립터 `fd`에서 디렉터리 엔트리를 읽습니다. `fd`는 반드시 디렉터리를 나타내야 합니다. 성공하면 널 종료된 파일 이름을
> `name`에 저장하고 true를 반환합니다. `name`에는 `READDIR_MAX_LEN + 1`바이트를 담을 공간이 있어야 합니다. 디렉터리에
> 남은 엔트리가 없으면 false를 반환합니다.
>
> `readdir`는 `.`과 `..`을 반환하면 안 됩니다.
> 디렉터리가 열려 있는 동안 내용이 바뀌는 경우, 일부 엔트리가 전혀 읽히지 않거나 여러 번 읽히는 것은 허용됩니다.
> 그렇지 않다면 각 디렉터리 엔트리는 순서에 상관없이 한 번씩만 읽혀야 합니다.
>
> `READDIR_MAX_LEN`은 `lib/user/syscall.h`에 정의되어 있습니다. 파일 시스템이 기본 파일 시스템보다
> 더 긴 파일 이름을 지원한다면, 이 값을 기본값 14에서 늘려야 합니다.

---

```c
bool isdir (int fd);
```

> `fd`가 디렉터리를 나타내면 true를, 일반 파일을 나타내면 false를 반환합니다.

---

```c
int inumber (int fd);
```

> `fd`와 연결된 inode의 inode 번호를 반환합니다. `fd`는 일반 파일일 수도 있고 디렉터리일 수도 있습니다.
>
> inode 번호는 파일이나 디렉터리를 영속적으로 식별합니다. 파일이 존재하는 동안에는 유일합니다. Pintos에서는 inode의 sector 번호를 inode 번호로 사용하기에 적합합니다.

### 소프트 링크

**Pintos에 소프트 링크 메커니즘을 구현하세요.** 
소프트 링크(soft link, 즉 symbolic link)는 다른 파일이나 디렉터리를 가리키는 의사 파일 객체입니다. 이 파일에는 가리키는 파일의 경로 정보가 절대 경로나 상대 경로 형태로 들어 있습니다. 다음 상황을 가정해 봅시다.

```
/
├── a
│   ├── link1 -> /file
│   │
│   └── link2 -> ../file
└── file
```

`/a` 안의 `link1`이라는 소프트 링크는 `/file`을 가리키며(절대 경로), `/a` 안의 `link2`는 `../file`을 가리킵니다(상대 경로).
`link1` 또는 `link2` 파일을 읽는 것은 `/file`을 읽는 것과 같습니다.

---

```c
int symlink (const char *target, const char *linkpath);
```

> 문자열 `target`을 담고 있는 `linkpath`라는 이름의 심볼릭 링크를 만듭니다.
> 성공하면 0을 반환하고, 그렇지 않으면 -1을 반환합니다.
