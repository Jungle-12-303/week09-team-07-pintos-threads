# 05_synchronization

### 동기화

제공된 파일 시스템은 외부 동기화를 요구합니다. 즉, 한 번에 오직 하나의 스레드만 파일 시스템 코드 안에서 실행되도록 호출자가 보장해야 합니다. 여러분의 제출물은 외부 동기화가 필요 없는, 더 세분화된 동기화 전략을 채택해야 합니다. 가능한 범위에서는 서로 독립적인 대상에 대한 연산도 서로 독립적으로 수행되어, 서로 기다릴 필요가 없어야 합니다.

서로 다른 캐시 블록에 대한 연산은 독립적이어야 합니다. 특히 특정 블록에 I/O가 필요할 때, I/O가 필요하지 않은 다른 블록에 대한 연산은 그 I/O가 끝날 때까지 기다리지 않고 진행될 수 있어야 합니다.

여러 프로세스가 하나의 파일에 동시에 접근할 수 있어야 합니다. 하나의 파일에 대한 여러 read는 서로를 기다리지 않고 완료될 수 있어야 합니다. 파일을 쓰는 동작이 파일 확장을 수반하지 않는다면, 여러 프로세스가 하나의 파일에 동시에 write할 수도 있어야 합니다. 한 프로세스가 파일을 쓰는 동안 다른 프로세스가 그 파일을 읽는 경우, 읽기 결과는 쓰기가 전혀 반영되지 않았거나, 전부 반영되었거나, 일부만 반영된 상태여도 됩니다. (하지만 `write` 시스템 콜이 호출자에게 반환된 이후의 모든 독자는 반드시 그 변경을 보아야 합니다.) 마찬가지로 두 프로세스가 동시에 파일의 같은 부분에 write하면, 각자의 데이터가 뒤섞여 기록될 수 있습니다.

반면, 파일을 확장하고 그 새 구간에 데이터를 쓰는 동작은 원자적이어야 합니다. 프로세스 A와 B가 같은 파일을 열고 있고 둘 다 EOF 위치에 있다고 가정해 봅시다. 이때 A가 read하고 B가 동시에 write한다면, A는 B가 쓴 내용의 전부, 일부, 또는 아무것도 읽지 못할 수 있습니다. 하지만 A가 B가 쓴 것 이외의 데이터를 읽어서는 안 됩니다. 예를 들어 B의 데이터가 전부 0이 아닌 바이트라면, A가 0을 보아서는 안 됩니다.

서로 다른 디렉터리에 대한 연산은 동시에 이루어져야 합니다. 같은 디렉터리에 대한 연산은 서로를 기다려도 됩니다.

동기화가 필요한 것은 여러 스레드가 공유하는 데이터뿐이라는 점을 기억하세요. 기본 파일 시스템에서 `struct file`과 `struct dir`은 각각 하나의 스레드만 접근합니다.
